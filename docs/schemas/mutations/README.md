# Mutation JSON Schemas

This directory contains JSON Schema files for every `Mutation` variant in
`trilithon-core`.  The schemas are generated by the `gen_mutation_schemas`
binary and must be committed alongside the source code so that drift is caught
by CI.

## Regenerating schemas

```bash
cargo run -p trilithon-core --bin gen_mutation_schemas
```

Or via the `just` target:

```bash
just check-schemas
```

## Schema format

The root `Mutation.json` uses JSON Schema Draft-07 with a `oneOf` / tagged
union pattern driven by the `"kind"` discriminant field (matching the `#[serde(tag = "kind")]`
attribute on the Rust enum).

Each per-variant file is a stub that carries:
- `"title"` — the variant name
- `"description"` — human-readable summary
- `"$ref"` — reference into `Mutation.json`
- `"x-variant"` — machine-readable variant tag

## CI drift check

The `just check-schemas` recipe runs the generator and then asserts that the
committed schemas match the freshly generated output via `git diff --exit-code`.
This prevents schema drift from accumulating silently.
